# Bilingual & Encoding Rules

## Language Policy
- Office communications → Hebrew only
- Hebrew respondents → Hebrew only
- English respondents → bilingual (EN first)
- Hebrew email subjects must start with a Hebrew character (not emoji) to prevent RTL reversal
- Email design rules: `docs/email-design-rules.md`

## User ↔ Agent Language
- User may write in Hebrew → agent ALWAYS responds in English
- All docs and code in English

## Hebrew Encoding Safety
If Hebrew appears garbled (`×× ×™×©×•×¨`) or has replacement chars (`�`) — **STOP immediately**. Do not proceed. Flag to user.
