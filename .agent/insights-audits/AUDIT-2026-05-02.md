## Section 2 — Wins (rules that worked)

1. **"Don't assume bug" + "Intentional Design Decisions"** (project CLAUDE.md, Communication Rules + Intentional Design Decisions sections)
   → Misunderstood Request **−48% / msg**. Agent now asks before classifying reports as bugs.
2. **"Be Surgical" cost-optimization rules** (project CLAUDE.md, Cost Optimization Rules)
   → File Too Large **−58% / msg**. Use of `sed -n`, `grep -n`, `head` over full reads is sticking.
3. **Session-start rules + design-log discipline** (`/design-log`, INDEX.md, Phase C plan-mode gate)
   → Wrong Approach **−25% / msg**. Plan-mode-first prevents wrong-direction starts.
4. **Throughput & quality:** Fully-achieved 46% → 64%; median 49s → 34s — combined effect of skills routing + memory recall + cleaner planning.
5. **n8n-safety rules:** Missed-Workflow events flat at 1 despite 2× message volume — `n8n_*` MCP rules + `n8n-mcp` skill are effective.

---

## Section 3 — Regressions / new friction

1. **Buggy Code +13% / msg.** First-pass fixes miss duplicate render paths. Concrete incidents:
   - Timestamp bug fixed in admin panel but not on AI-review page.
   - Queue stale-state required two follow-up commits.
   Root cause: agent edits the first matching site and stops; doesn't grep parallel surfaces (admin/, client-portal/, n8n/).
2. **Command Failed +35% / msg.** Bash/wrangler edge cases dominate:
   - Cleanup script `--is-ancestor origin/main` matched fresh zero-commit branches → **near-wipe of an active worktree** (2026-05-02).
   - Wrangler autoconfig hijacking deploys (already memorialized but still firing).
3. **User Rejected mid-execution +94% / msg.** Two patterns:
   - User interrupting to add requirements (scope shift mid-implementation).
   - Agent starting work without restating problem + needed inputs (IDs, URLs, creds).
4. **File Changed (43 events).** New tool-error category, almost certainly multi-clauding (26% of msgs were in parallel sessions). Agent edits a file, file mtime advanced by another session, edit fails or clobbers.
5. **Sentiment dipped 2.5 pp** (90.0% → 87.5%). Tracks the regressions above.

---

## Section 4 — New patterns observed

- **Multi-clauding is now load-bearing** (26% of msgs). Project rules already cover this in *Multi-Tab Safety* but enforcement is per-session and reactive — no PostToolUse-level detection of cross-session collisions.
- **Airtable typecast / field-name guesses** are an emerging silent-failure class — small (6 events) but each one was a write that "succeeded" without effect.
- **Tooling Friction** — terminal config, wrangler invocations, cleanup scripts. Pattern: destructive-ish shell scripts run without dry-run.
- **Top wins are reinforcement-shaped:** "silent UI refresh after DB mutation" already in MEMORY.md as a positive rule; promoting it from implicit to explicit in project CLAUDE.md would strengthen.
