## 3. What's Still Friction

### F1: Wrong Initial Approach (26 events — 43% of friction)

**Root causes:**
1. Claude doesn't pull latest code before starting
2. Claude starts in wrong directory
3. Claude makes wrong assumptions about architecture
4. Claude presents output inline instead of saving to files

**Classification:**
- Missing CLAUDE.md rule: "Always git pull before work"
- Missing CLAUDE.md rule: "Always verify git repo directory"
- Missing CLAUDE.md rule: "Save audit/plan output to files, not inline"
- Possible hook: auto-pull on session start

---

### F2: Buggy Code (15 events — 25% of friction)

**Root causes:**
1. CSP directives required 3+ iterations (missed Google Fonts, font-src, inline handlers)
2. API filter logic failed with null/undefined fields
3. Encoding issues not caught before deployment

**Classification:**
- Workflow/habit issue: No pre-implementation checklist for edge cases
- Missing CLAUDE.md rule: For CSP changes, audit ALL origins first
- Missing CLAUDE.md rule: Test from user/browser perspective, not just API

---

### F3: Misunderstood Request (12 events — 20% of friction)

**Root causes:**
1. Flagging intentional design choices as bugs (disabled chat widget, 2-day limit, email notifications)
2. Confusing which error logs user is pointing to
3. Assuming wrong tools/models

**Classification:**
- Project documentation gap: No "Intentional Design Decisions" section
- Missing CLAUDE.md rule: "Don't assume user-reported behavior is a bug — ask first"

---

### F4: Missed Active Workflows (1 critical event)

**Root cause:** Claude modified an inactive duplicate workflow while the replacement was the live one — PII continued leaking in production.

**Classification:**
- Missing CLAUDE.md rule: "List ALL matching workflows and check execution counts before modifying any"
- Possible skill: `/security-audit` with mandatory workflow discovery step

---

### F5: Planning+Implementation Sessions Run Out of Steam

**Root cause:** Sessions that combine both planning AND implementation hit "mostly achieved" rather than "fully achieved." The 36% "mostly achieved" rate often stems from this pattern.

**Classification:**
- Workflow/habit issue: Split planning and implementation into separate sessions
- Already partially addressed by design-log workflow, but not consistently enforced

---

### F6: Tool Errors (504 total)

**Notable:**
- 93 User Rejected actions — Claude attempting things user doesn't want
- 35 File Too Large — reading oversized files without checking first
- 33 File Not Found — wrong paths
- 162 Command Failed — bash commands failing

**Classification:**
- Missing CLAUDE.md rule already exists for "Check file size before reading" — may need reinforcement
- File Not Found suggests wrong directory assumptions (ties to F1)

## 4. Actionable Improvements

### Improvement 1: Add "Workflow Rules" to Project CLAUDE.md

**Location:** Project-local `CLAUDE.md`
**Section:** Add under `## Operating Mode`

```markdown
## Workflow Rules
1. Always run `git pull` before starting any work. Never analyze or edit without pulling first.
2. Verify git repo directory with `git rev-parse --show-toplevel` before any git operations.
3. When producing findings, audits, or plans — save to files (design logs, plan files), not inline in chat.
4. For n8n tasks: list ALL matching workflows and check recent execution counts to identify which is active before modifying any.
```

**Friction addressed:** F1 (wrong approach), F4 (missed workflows)

---

### Improvement 2: Add "Communication Rules" to Project CLAUDE.md

**Location:** Project-local `CLAUDE.md`
**Section:** New section

```markdown
## Communication Rules
1. When the user reports a behavior, do NOT assume it's a bug. Ask before classifying — it may be an intentional design choice.
2. Test changes from the end-user/browser perspective, not just API level. Provide clickable URLs and describe what user should see.
```

**Friction addressed:** F3 (misunderstood request), F2 (buggy code from API-only testing)

### Improvement 4: Add CSP/Security Rule to Project CLAUDE.md

**Location:** Project-local `CLAUDE.md`
**Section:** Under existing security context
