## Root Cause

**Multiple mappings generating the same document type**, some without proper `detailsField`:

### Form 106
1. ✅ `employment_employers_list` (line 336-348) - `perItem: true`, `detailsField: "issuer_name"` → Works correctly
2. ❌ `employment_was_employed` (line 324-334) - `perItem: false`, `detailsField: null` → Creates duplicate with `{employer}` placeholder

### Form 106 Spouse
1. ✅ `employment_spouse_employers_list` (line 362-374) - `perItem: true`, `detailsField: "issuer_name"` → Works correctly
2. ❌ `employment_spouse_was_employed` (line 350-360) - `perItem: false`, `detailsField: null` → Creates duplicate

### Form 867
1. ✅ `investments_securities_institutions` (line 499-510) - `perItem: true`, `detailsField: "issuer_name"` → Works
2. ❌ `investments_securities` (line 486-496) - `perItem: false`, `detailsField: null` → Creates duplicate
3. ✅ `investments_deposits_banks` (line 525-536) - `perItem: true`, `detailsField: "issuer_name"` → Works
4. ❌ `investments_deposits` (line 512-522) - `perItem: false`, `detailsField: null` → Creates duplicate

### Crypto Report
- ❌ `investments_crypto` (line 538-548) - Generates `crypto_report` but has no `detailsField`
- ❌ `investments_crypto_details` (line 550-561) - Doesn't generate documents at all

## The Fix

**Strategy**: Parent yes/no questions should ONLY trigger linked questions, NOT generate documents themselves.

### Changed Mappings (questionnaire-mapping.js)

**1. employment_was_employed (line 324):**
```javascript
documents: [],  // Documents generated by employment_employers_list
```

**2. employment_spouse_was_employed (line 350):**
```javascript
documents: [],  // Documents generated by employment_spouse_employers_list
```

**3. investments_securities (line 486):**
```javascript
documents: [],  // Documents generated by investments_securities_institutions
```

**4. investments_deposits (line 512):**
```javascript
documents: [],  // Documents generated by investments_deposits_banks
```

**5. investments_crypto (line 538):**
```javascript
documents: [],  // Documents generated by investments_crypto_details
```

**6. investments_crypto_details (line 550):**
```javascript
{
  id: "investments_crypto_details",
  type: "list",  // Changed from "text" to "list"
  documents: ["crypto_report"],
  perItem: true,  // Generate one document per platform
  detailsField: "platform",  // Fill {platform} placeholder
  // ...
}
```
